# 🔄 更新机制说明

本文档说明应用的自动更新功能。

## 概述

应用内置自动更新功能，通过 GitHub Releases 实现：

1. **版本检测**：在设置中点击"检查更新"，或应用启动时自动检测
2. **信息获取**：通过 GitHub API 获取最新版本信息
3. **更新提示**：显示更新内容
4. **APK 下载**：自动下载最新 APK
5. **安装更新**：调用系统安装程序完成更新

## 使用方法

### 手动检查更新

1. 打开应用，进入"设置"页面
2. 点击"检查更新"
3. 如果有新版本，会显示更新内容和下载选项
4. 下载完成后点击安装

### 自动更新说明

- 应用会定期检查更新（目前需要手动触发）
- 更新源：https://github.com/YanceyQian/res-downloader-android/releases
- 建议从 GitHub Releases 下载最新版本安装

## 远程配置更新

除了应用版本更新外，系统还支持远程配置更新，用于热更新平台解析规则：

### 配置更新特点

| 特性 | 应用更新 | 配置更新 |
|------|---------|---------|
| 更新内容 | 代码变更 | 平台规则/API |
| 更新频率 | 版本发布时 | 按需更新 |
| 用户操作 | 需要安装 | 自动获取 |
| 安全性 | APK 签名 | 配置签名验证 |

### 配置更新流程

1. 后台检查配置版本（应用启动时自动检查）
2. 下载新配置（通过 HTTPS 安全传输）
3. 验证配置签名（确保配置来源可信）
4. 应用新配置（无需重启，自动生效）

### 配置更新优势

- **快速响应**：平台 API 变更后，无需等待版本发布即可修复
- **节省流量**：配置数据小，下载快速
- **无缝体验**：用户无感知，静默更新
- **安全性高**：配置签名验证，防止恶意篡改

### 紧急更新机制

当平台 API 突然变更（如抖音/微信更新导致解析失败）时：

1. **收到反馈**：用户反馈某个平台解析失败
2. **问题确认**：确认是 API 变更而非临时故障
3. **紧急修复**：在配置服务器发布紧急修复配置
4. **用户获取**：应用在 24 小时内自动获取新配置
5. **验证效果**：监控修复效果，确认问题解决

### 配置版本说明

配置版本与应用版本相互独立：

- **配置版本**：管理平台解析规则，当前版本可在设置中查看
- **应用版本**：管理代码功能，通过 GitHub Releases 发布

详细设计请参考：[PLUGIN_SYSTEM_FEASIBILITY.md](PLUGIN_SYSTEM_FEASIBILITY.md)

## APK 文件格式

应用会自动识别以下格式的 APK：

- `res-downloader_v{VERSION}_android.apk`
- 例如：`res-downloader_v1.0.0_android.apk`
