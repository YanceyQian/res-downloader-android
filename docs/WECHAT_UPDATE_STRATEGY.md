## 1. 问题背景

当微信版本更新时，可能导致以下问题：

| 问题类型 | 原因 | 表现 |
|----------|------|------|
| **API 返回错误码** | 接口签名验证变更 | HTTP 400/401/403 |
| **数据格式变化** | JSON 结构更新 | 解析失败，返回 null |
| **加密方式变化** | 解密参数变更 | 解密失败，播放异常 |
| **接口下线** | 旧 API 被废弃 | 404 Not Found |
| **签名验证失败** | 签名算法变更 | 校验不通过 |

## 2. 错误分类体系

### 2.1 错误类型枚举

```kotlin
enum class WechatErrorType(
    val code: Int,
    val title: String,
    val userMessage: String,
    val isApiRelated: Boolean,      // 是否为 API 相关错误
    val requiresAppUpdate: Boolean  // 是否需要更新 APP
) {
    // API 相关错误（可能需要更新 APP）
    API_RET_ERROR(1001, "API 返回错误", "微信 API 返回了错误，请更新到最新版本", true, true),
    API_NETWORK_ERROR(1002, "网络请求失败", "网络连接失败，请检查网络", false, false),
    API_TIMEOUT(1003, "请求超时", "请求超时，请重试", false, false),
    API_INVALID_RESPONSE(1004, "响应格式错误", "微信响应格式已更新，请更新到最新版本", true, true),
    API_AUTH_FAILED(1005, "认证失败", "微信认证失败，可能需要更新版本", true, true),
    API_QUOTA_EXCEEDED(1006, "配额超限", "请求过于频繁，请稍后再试", false, false),

    // 解密相关错误
    DECRYPT_KEY_MISSING(2001, "密钥缺失", "视频解密密钥获取失败，请更新到最新版本", true, true),
    DECRYPT_KEY_INVALID(2002, "密钥无效", "视频解密密钥已失效，请更新到最新版本", true, true),
    DECRYPT_FAILED(2003, "解密失败", "视频解密失败，可能是微信版本不兼容", true, true),

    // 链接解析错误
    URL_INVALID(3001, "链接无效", "视频链接格式不正确", false, false),
    URL_SHORT_RESOLVE_FAILED(3002, "短链接解析失败", "无法解析视频链接", false, false),
    VIDEO_ID_MISSING(3003, "视频 ID 缺失", "无法获取视频 ID，请检查链接", false, false),

    // 下载相关错误
    DOWNLOAD_FAILED(4001, "下载失败", "视频下载失败，请重试", false, false),
    DOWNLOAD_INTERRUPTED(4002, "下载中断", "下载被中断，请重试", false, false),
    STORAGE_FULL(4003, "存储空间不足", "存储空间不足，请清理后重试", false, false),

    // 未知错误
    UNKNOWN(9999, "未知错误", "发生未知错误，请更新到最新版本", true, true);
}
```

### 2.2 错误分类逻辑

```
错误分类树：
├── 网络问题（isApiRelated: false, requiresAppUpdate: false）
│   ├── 网络连接失败
│   ├── 请求超时
│   └── 下载中断
│
├── API 问题（isApiRelated: true, requiresAppUpdate: true）
│   ├── API 返回错误码
│   ├── 响应格式变化
│   ├── 认证失败
│   └── 配额超限
│
├── 解密问题（isApiRelated: true, requiresAppUpdate: true）
│   ├── 密钥缺失
│   ├── 密钥无效
│   └── 解密失败
│
└── 用户问题（isApiRelated: false, requiresAppUpdate: false）
    ├── 链接格式错误
    ├── 存储空间不足
    └── 视频 ID 缺失
```

## 3. 检测机制

### 3.1 连续失败检测

当用户在短时间内（10 分钟内）连续遇到 API 相关错误达到阈值次数（默认 3 次），系统判定为 **API 可能已失效**：

```kotlin
// 连续 API 错误次数阈值，超过则判定为 API 失效
private const val API_FAILURE_THRESHOLD = 3

// 检查是否达到 API 失效阈值
if (apiErrorCount >= API_FAILURE_THRESHOLD) {
    _isApiLikelyBroken.value = true
}
```

### 3.2 错误检测流程

```
用户操作
    │
    ▼
解析微信视频链接
    │
    ├─── 成功 ────────▶ 显示资源
    │
    └─── 失败 ──▶ 捕获异常
                    │
                    ▼
            错误类型分类
            (fromException)
                    │
                    ├─── isApiRelated = false ──▶ 提示用户检查网络
                    │
                    └─── isApiRelated = true ──▶ 增加 API 错误计数
                                                │
                                                ▼
                                        检查是否达到阈值
                                                │
                                                ├─── 未达到 ──▶ 提示"请更新到最新版本"
                                                │
                                                └─── 达到 ──▶ 显示 API 失效警告
```

### 3.3 错误自动上报

错误信息会自动保存到本地缓存，用于后续分析：

```kotlin
data class WechatErrorDetail(
```

## 4. 提醒策略

### 4.1 提醒级别

| 级别 | 触发条件 | 提醒方式 | 用户操作 |
|------|----------|----------|----------|
| **L1 提示** | 单次 API 错误 | Toast/Snackbar | 查看错误信息 |
| **L2 警告** | 连续 3 次 API 错误 | 顶部警告条 | 检查更新 |
| **L3 弹窗** | 发现新版本 | 对话框 | 立即更新/稍后 |

## 5. 更新跟进机制

### 5.1 公告系统

使用 `announcement.json` 文件维护公告，支持版本范围、时间范围等配置：

```json
{
  "id": "wechat-api-update-2026-05",
  "title": "微信视频号接口更新通知",
  "content": "微信已于近期更新了视频号 API，部分用户可能出现抓取失败。",
  "type": "WARNING",
  "fixedInVersion": "2.2.0",
  "minAppVersion": "2.2.0",
  "maxAppVersion": "2.3.0",
  "startTime": "2026-05-01T00:00:00Z",
  "endTime": "2026-06-01T00:00:00Z",
  "actionUrl": "https://github.com/YanceyQian/res-downloader-android/releases",
  "actionText": "立即更新"
}
```

**关键字段**：
- `fixedInVersion`: 标注在哪个版本已修复此问题
- `actionUrl`: 用户点击按钮时打开的链接

### 5.2 公告类型

| 类型 | 颜色 | 用途 |
|------|------|------|
| `INFO` | 灰色 | 普通信息 |
| `WARNING` | 橙色 | 警告（如微信版本更新） |
| `URGENT` | 红色 | 紧急（如 API 完全失效） |
| `UPDATE` | 蓝色 | 更新提示 |

### 5.3 版本检查流程

```
APP 启动
    │
    ▼
检查公告系统（从 GitHub 读取 announcement.json）
    │
    ├─── 有公告且适用 ──▶ 显示公告横幅
    │
    └─── 无公告 ──▶ 静默
    │
    ▼
后台检查 GitHub Releases
    │
    ├─── 发现新版本 ──▶ 显示更新提醒对话框
    │
    └─── 无新版本 ──▶ 结束
```

## 6. 维护计划

### 6.1 快速响应流程

```
微信版本更新
    │
    ▼
用户反馈（GitHub Issues / 应用商店评论）
    │
    ▼
确认问题（验证 API 是否确实失效）
    │
    ▼
发布紧急公告（更新 announcement.json）
    │
    ▼
分析新 API 变更（适配新版本）
    │
    ▼
更新代码并发布新版本
    │
    ▼
更新公告为"已修复"
```

### 6.2 开发者任务清单

当收到微信 API 失效反馈时，开发者应：

1. **确认问题**（5 分钟）
   - [ ] 验证用户提供的信息
   - [ ] 在本地测试是否确实失效
   - [ ] 记录错误类型和微信版本

2. **发布公告**（10 分钟）
   - [ ] 更新 `announcement.json`
   - [ ] 推送到 GitHub
   - [ ] 通知用户已有公告

3. **修复问题**（2-24 小时，取决于复杂度）
   - [ ] 分析新 API 的变化
   - [ ] 更新代码
   - [ ] 测试修复

4. **发布更新**（30 分钟）
   - [ ] 发布新版本 APK
   - [ ] 更新 release notes
   - [ ] 更新公告为"已修复"

### 6.3 预期响应时间

| 问题类型 | 预期响应时间 |
|----------|--------------|
| 单用户反馈 | 24-48 小时内确认 |
| 多用户反馈 | 2-4 小时内确认 |
| 公告发布 | 确认后 1 小时内 |
| 临时修复 | 确认后 24 小时内 |
| 完整修复 | 视复杂度而定 |

## 5. 降级方案

当自动抓取失败且更新也无法解决问题时，系统会提供降级方案。

### 5.1 降级方案流程

```
用户抓取失败
    │
    ▼
连续失败 3 次 → 显示降级方案
    │
    ▼
用户选择方案
    │
    ├─── 分享链接方式 ──▶ 引导用户手动分享链接
    ├─── 反馈 ──▶ 打开反馈对话框
    └─── 其他工具 ──▶ 提示其他解决方案
```

### 5.2 降级方案选项

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| **分享链接** | 用户手动分享链接到应用 | API 完全失效，但链接仍可解析 |
| **录屏保存** | 使用手机录屏功能 | 视频无法下载时 |
| **其他工具** | 建议使用其他支持工具 | 临时替代方案 |
| **反馈** | 提交问题反馈 | 帮助开发者定位问题 |

## 6. 反馈机制

### 6.1 反馈收集

当用户提交反馈时，系统会收集以下信息：

```kotlin
data class UserFeedback(
    val title: String,              // 反馈标题
    val description: String,        // 用户描述的问题
    val appVersion: String,         // 应用版本
    val wechatVersion: String?,     // 微信版本（如果能获取）
    val errorCount: Int,            // 错误次数
    val lastErrorType: String?,     // 最后错误类型
    val timestamp: Long             // 提交时间
)
```

### 6.2 反馈模板

系统会自动生成反馈模板：

```markdown
## 问题描述
（请描述您遇到的问题）

## 操作步骤
1. 打开微信视频号
2. 点击某个视频
3. 点击分享 -> 复制链接
4. 粘贴到本应用中
5. 点击下载

## 错误信息
- 错误类型: API_RET_ERROR
- 错误消息: API returned error code: -1
- 应用版本: 2.1.0
- 微信版本: 8.0.50

## 其他信息
（可选）提供视频链接（非必须）：
```

### 6.3 反馈使用方式

1. **本地缓存**：反馈首先保存到本地 `wechat_feedback.json`
2. **导出方式**：
   - 通过 ADB 导出：`adb pull /data/data/com.resdownloader/files/wechat_feedback.json`
   - 用户直接复制反馈内容到 GitHub Issues

## 7. 公告系统

### 7.1 公告配置

公告文件 `announcement.json` 存储在 GitHub 仓库的根目录：

```json
{
  "id": "wechat-api-update-2026-05",
  "title": "微信视频号接口更新通知",
  "content": "微信已于近期更新了视频号 API，部分用户可能出现抓取失败。",
  "type": "WARNING",
  "fixedInVersion": "2.2.0",
  "startTime": "2026-05-01T00:00:00Z",

  "actionUrl": "https://github.com/YanceyQian/res-downloader-android/releases",

}
```

### 7.2 公告字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | String | ✅ | 唯一标识，用于去重 |
| `title` | String | ✅ | 公告标题 |
| `content` | String | ✅ | 公告内容，支持 `\n` 换行 |
| `type` | Enum | ✅ | 类型：INFO/WARNING/URGENT/UPDATE |
| `fixedInVersion` | String | ❌ | 标注在哪个版本已修复 |
| `minAppVersion` | String | ❌ | 最低适用版本 |
| `maxAppVersion` | String | ❌ | 最高适用版本 |
| `startTime` | ISO Date | ❌ | 公告开始时间 |
| `endTime` | ISO Date | ❌ | 公告结束时间 |
| `actionUrl` | String | ❌ | 点击按钮打开的链接 |
| `actionText` | String | ❌ | 按钮文字 |

### 7.3 公告更新流程

```
微信版本更新
    │
    ▼
开发者确认问题
    │
    ▼
创建/更新 announcement.json
    │
    ├─── 已修复 ──▶ 添加 fixedInVersion

    │                   ▼
    │              设置 minAppVersion = fixedInVersion
    │
    └─── 未修复 ──▶ 不添加 fixedInVersion
                        │
                        ▼
                   提示用户等待
```

### 8.2 开发者任务清单

1. **确认问题**（5 分钟）
   - [ ] 验证用户提供的信息
   - [ ] 在本地测试是否确实失效
   - [ ] 记录错误类型和微信版本

2. **发布公告**（10 分钟）
   - [ ] 更新 `announcement.json`
   - [ ] 推送到 GitHub
   - [ ] 通知用户已有公告

3. **修复问题**（2-24 小时，取决于复杂度）
   - [ ] 分析新 API 的变化
   - [ ] 更新代码
   - [ ] 测试修复

4. **发布更新**（30 分钟）
   - [ ] 发布新版本 APK
   - [ ] 更新 release notes
   - [ ] 更新公告添加 `fixedInVersion`

### 8.3 预期响应时间

| 问题类型 | 预期响应时间 |
|----------|--------------|
| 单用户反馈 | 24-48 小时内确认 |
| 多用户反馈 | 2-4 小时内确认 |
| 公告发布 | 确认后 1 小时内 |
| 临时修复 | 确认后 24 小时内 |
| 完整修复 | 视复杂度而定 |

### 8.4 降级方案状态管理

系统会自动管理降级方案的状态：

```kotlin
sealed class FallbackState {
    object None : FallbackState()                              // 无降级需要
    data class SuggestShareLink(val message: String) : FallbackState()  // 建议分享链接
    data class SuggestAlternative(val alternatives: List<String>) : FallbackState()  // 建议其他方案
    data class AllFailed(val feedbackUrl: String) : FallbackState()  // 所有方案都失败
}
```
