#### Step 2.2: decodeKey 自动提取 ⭐ [12h] 【核心】

**方案 A：Frida Gadget 集成（推荐）** ⭐⭐⭐⭐⭐
```
无需 Root！用户只需安装打包好的 APK

原理：
1. 将 Frida Gadget（动态 instrumentation 工具）嵌入 APK
2. Gadget 通过 Unix Socket 与主应用通信
3. Hook 微信的 `get media()` 方法
4. 自动提取 decodeKey 和视频 URL

优势：
✅ 无需 Root/Xposed
✅ 透明、无感知
✅ 自动捕获所有视频请求
✅ 兼容微信多版本

实施步骤：
1. 下载 Frida Gadget ARM64 库
2. 将 Gadget.so 放入 app/src/main/jniLibs/
3. 创建 Gadget 加载服务
4. 实现 Unix Socket 通信
5. 编写 Hook 脚本（JavaScript）
6. 集成到主应用

Hook 脚本示例：
Java.perform(function() {
    var mediaClass = Java.use("com.tencent.mm.modelcdnlink");
    mediaClass.getMedia.implementation = function() {
        var result = this.getMedia.apply(this, arguments);
        // 发送数据到主应用
        send({"url": result.url, "decodeKey": result.decodeKey});
        return result;
    }
});
```

**方案 B：微信开放平台接口** ⭐⭐⭐⭐
```
利用微信官方能力，无需任何 Hook

原理：
1. 微信小程序有开放的数据获取接口
2. 通过 URL Scheme 唤起小程序特定页面
3. 使用 application: 协议获取分享信息

限制：
⚠️ 仅支持已配置的小程序
⚠️ 需要开发者资质
⚠️ 功能受限

实施步骤：
1. 注册微信开放平台账号
2. 申请移动应用权限
3. 使用 WXAPIFactory 获取数据
```

**方案 C：二维码扫描 + API 解析** ⭐⭐⭐⭐⭐
```
用户友好，适合普通用户

原理：
1. 提供二维码扫描入口
2. 解析视频号二维码获取 vid
3. 调用现有 API 获取视频信息

优势：
✅ 完全合规，无风险
✅ 无需任何特殊权限
✅ 兼容所有微信版本

实施步骤：
1. 集成 ZXing 扫码库
2. 解析视频号小程序码 URL
3. 提取 vid 参数
4. 调用视频详情 API
```

## 六、非 Root 用户解决方案（v2.2 新增）

### 6.1 方案概述

Android 版微信视频号抓取有多种方案，Frida Gadget 是成功率最高的可选增强方案：

| 方法 | 成功率 | 风险 | 用户操作 | 是否需要勾选 |
|------|--------|------|----------|--------------|
| **Frida Gadget** | ⭐⭐⭐⭐⭐ 95% | 中 | 安装即用 | ✅ 需要 |
| **分享链接 API** | ⭐⭐ 50% | 无 | 复制链接 | ❌ 不需要 |
| **VPN + 证书** | ⭐⭐⭐ 30% | 低 | 装证书 | ❌ 不推荐 |
| **二维码扫描** | ⭐⭐⭐ 40% | 无 | 扫码 | ❌ 不需要 |

### 6.2 推荐方案：Frida Gadget 集成 ⭐⭐⭐⭐⭐

**原理**：将 Frida 动态 instrumentation 工具嵌入 APK，用户安装后自动具备 Hook 能力

**优势**：
- ✅ 完全无需 Root
- ✅ 用户无感知，安装即用
- ✅ 自动捕获所有视频请求
- ✅ 兼容主流微信版本

**用户不勾选风险提示时的降级方案**：
- 自动切换到分享链接 API 方案
- 成功率降为 50%，但无封号风险
- 其他功能（抖音、B站等）完全不受影响

## 十二、Android 版与桌面版架构差异说明

### 12.1 核心差异

| 对比项 | 桌面版 (res-downloader) | Android 版 (本项目) |
|--------|------------------------|---------------------|
| **技术栈** | Go + Wails | Kotlin + Jetpack Compose |
| **代理方式** | MITM 代理 + 插件系统 | VPN Service + API 解析 |
| **平台支持** | 自动支持所有平台（域名规则） | 需要为每个平台单独实现 |
| **资源获取** | 在目标 APP 中操作即可自动抓取 | 复制分享链接 或 开启代理 |
| **HTTPS 拦截** | 需要安装证书/代理转发 | VPN Service 可直接拦截 |
| **用户操作** | 完全透明，无需额外操作 | 需要主动复制链接 |
| **使用场景** | 电脑端 | 移动端随时随地 |

### 12.2 Android 版限制原因

Android 版**无法像桌面版那样自动支持所有平台**，原因如下：

1. **架构限制**
   - 桌面版：通过 MITM 代理拦截所有 HTTP/HTTPS 流量
   - Android 版：VPN Service 只能拦截本机流量，无法自动识别新平台

2. **平台解析**
   - 桌面版：通过域名规则自动匹配，无需额外配置
   - Android 版：需要为每个平台单独实现 API 解析逻辑

3. **用户操作**
   - 桌面版：用户无感知，资源自动抓取
   - Android 版：需要用户主动复制分享链接

### 12.3 Android 版解决方案

为了在 Android 端实现类似功能，我们采用了两种方式：

1. **分享链接解析（推荐）**
   - 用户复制分享链接
   - 应用解析链接获取资源地址
   - 无需开启代理，节省电量
   - 简单快捷

2. **代理抓取**
   - 开启 VPN 代理
   - 自动捕获应用中的资源
   - 适用于分享链接不支持的资源
   - 需要安装证书（可选）
