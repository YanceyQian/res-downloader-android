#### Step 2.2: decodeKey 自动提取 ⭐ [12h] 【核心】

**方案 A：Frida Gadget 集成（推荐）** ⭐⭐⭐⭐⭐
```
无需 Root！用户只需安装打包好的 APK

原理：
1. 将 Frida Gadget（动态 instrumentation 工具）嵌入 APK
2. Gadget 通过 Unix Socket 与主应用通信
3. Hook 微信的 `get media()` 方法
4. 自动提取 decodeKey 和视频 URL

优势：
✅ 无需 Root/Xposed
✅ 透明、无感知
✅ 自动捕获所有视频请求
✅ 兼容微信多版本

实施步骤：
1. 下载 Frida Gadget ARM64 库
2. 将 Gadget.so 放入 app/src/main/jniLibs/
3. 创建 Gadget 加载服务
4. 实现 Unix Socket 通信
5. 编写 Hook 脚本（JavaScript）
6. 集成到主应用

Hook 脚本示例：
Java.perform(function() {
    var mediaClass = Java.use("com.tencent.mm.modelcdnlink");
    mediaClass.getMedia.implementation = function() {
        var result = this.getMedia.apply(this, arguments);
        // 发送数据到主应用
        send({"url": result.url, "decodeKey": result.decodeKey});
        return result;
    }
});
```

**方案 B：微信开放平台接口** ⭐⭐⭐⭐
```
利用微信官方能力，无需任何 Hook

原理：
1. 微信小程序有开放的数据获取接口
2. 通过 URL Scheme 唤起小程序特定页面
3. 使用 application: 协议获取分享信息

限制：
⚠️ 仅支持已配置的小程序
⚠️ 需要开发者资质
⚠️ 功能受限

实施步骤：
1. 注册微信开放平台账号
2. 申请移动应用权限
3. 使用 WXAPIFactory 获取数据
```

**方案 C：二维码扫描 + API 解析** ⭐⭐⭐⭐⭐
```
用户友好，适合普通用户

原理：
1. 提供二维码扫描入口
2. 解析视频号二维码获取 vid
3. 调用现有 API 获取视频信息

优势：
✅ 完全合规，无风险
✅ 无需任何特殊权限
✅ 兼容所有微信版本

实施步骤：
1. 集成 ZXing 扫码库
2. 解析视频号小程序码 URL
3. 提取 vid 参数
4. 调用视频详情 API
```

**方案 D：VPN 流量捕获** ⭐⭐⭐
```
在 VPN 层捕获解密后的请求

前提条件：
1. 用户信任自签名 CA 证书
2. Android 7.0+ 需要配置 networkSecurityConfig

实施步骤：
1. 生成自签名 CA 证书
2. 配置 networkSecurityConfig
3. VPN 捕获 HTTPS 明文流量
4. 解析响应中的视频信息
```

## 六、非 Root 用户解决方案（新增）

### 6.1 推荐方案：Frida Gadget 集成 ⭐⭐⭐⭐⭐

**原理**：将 Frida 动态 instrumentation 工具嵌入 APK，用户安装后自动具备 Hook 能力

**优势**：
- ✅ 完全无需 Root
- ✅ 用户无感知，安装即用
- ✅ 自动捕获所有视频请求
- ✅ 兼容主流微信版本

**实施难度**：中高（开发者需要）
**用户体验**：⭐⭐⭐⭐⭐ 极简

### 6.2 备选方案：二维码扫描 + API ⭐⭐⭐⭐

**原理**：扫描视频号小程序码 → 解析 URL → 调用 API

**优势**：
- ✅ 完全合规，无风险
- ✅ 无需特殊权限
- ✅ 实现简单

**劣势**：
- ⚠️ 需要用户主动扫码
- ⚠️ 无法自动捕获

**适用场景**：普通用户首选方案

### 6.3 备选方案：VPN + 自签名证书 ⭐⭐⭐

**原理**：用户安装 CA 证书 → VPN 捕获 HTTPS 明文

**优势**：
- ✅ 通用抓包方案
- ✅ 可用于其他 App

**劣势**：
- ⚠️ 需要用户手动安装证书
- ⚠️ Android 7.0+ 部分 App 不生效

**适用场景**：技术用户
